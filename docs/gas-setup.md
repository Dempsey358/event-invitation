# Google Apps Script セットアップ手順

出欠回答をGoogleスプレッドシートに保存するためのセットアップ手順です。

## 1. Googleスプレッドシートの作成

1. [Google Sheets](https://sheets.google.com) を開く
2. **新しいスプレッドシート** を作成
3. 名前を「イベント出欠管理」などに変更

## 2. Apps Scriptを開く

1. メニュー → **拡張機能** → **Apps Script** をクリック
2. エディタが開く

## 3. スクリプトを貼り付け

1. デフォルトの `function myFunction() {}` を全て削除
2. `gas/Code.gs` の内容を全てコピー＆ペースト
3. **Ctrl+S** で保存（プロジェクト名は「RSVP」などに変更）

## 4. Webアプリとしてデプロイ

1. 右上の **「デプロイ」** ボタン → **「新しいデプロイ」**
2. 左上の歯車アイコン → **「ウェブアプリ」** を選択
3. 設定：
   - **説明**: 出欠回答受付
   - **次のユーザーとして実行**: **自分**
   - **アクセスできるユーザー**: **全員**
4. **「デプロイ」** をクリック
5. **アクセスを承認** を求められたら、自分のGoogleアカウントで許可
6. 表示された **ウェブアプリのURL** をコピー

> ⚠️ 「このアプリは確認されていません」の警告が出る場合：
> 「詳細」→「（プロジェクト名）に移動」→「許可」

## 5. config.json に反映

コピーしたURLを `config.json` の `gasUrl` に設定：

```json
{
  "gasUrl": "https://script.google.com/macros/s/xxxxxxxx/exec"
}
```

## 6. 動作確認

1. ブラウザでWebアプリのURLにアクセス → `{"status":"ok","message":"RSVP API is running"}` が表示されればOK
2. 招待状サイトから出欠回答を送信 → スプレッドシートに行が追加される

## トラブルシューティング

| 問題                   | 解決策                                                             |
| ---------------------- | ------------------------------------------------------------------ |
| 送信しても保存されない | GASのURL が正しいか確認。デプロイが「全員」に公開されているか確認  |
| CORSエラー             | `mode: 'no-cors'` が設定されているか確認（デフォルトで設定済み） |
| 更新が反映されない     | スクリプト変更後は**新しいデプロイ** を作成する必要がある    |
